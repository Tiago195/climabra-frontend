Feature nova a adicionar: Exceções
O provider precisa poder bloquear:

Dia inteiro — feriados (Natal, Ano Novo)
Intervalo de dias inteiros — férias (1 ou 2 semanas seguidas)
Faixa horária num dia específico — consulta médica das 14h às 16h amanhã
Faixa horária aplicada num intervalo de dias — ex: 15-22/01 todos os dias das 14h às 17h não atendo
A faixa horária, quando definida, aplica-se a todos os dias do intervalo igualmente. Se o provider precisa de horários diferentes em dias diferentes, ele cria múltiplas exceções.

Contrato com a API (você consome)
Tipos TypeScript a usar/criar em src/services/availability.ts
export interface IExceptionResponse {
  id: string;
  startDate: string;          // formato "YYYY-MM-DD"
  endDate: string;            // formato "YYYY-MM-DD" (inclusive; pode ser igual a startDate)
  startTime: string | null;   // formato "HH:mm:ss" — NULL significa "dia inteiro"
  endTime: string | null;     // formato "HH:mm:ss" — NULL significa "dia inteiro"
  reason: string | null;      // motivo opcional (ex: "Férias", "Natal")
}

export interface IExceptionPayload {
  startDate: string;          // obrigatório
  endDate: string;            // obrigatório
  startTime?: string;         // opcional — se omitidos juntos, é dia inteiro
  endTime?: string;
  reason?: string;
}
Regra: startTime e endTime aparecem juntos ou ausentes juntos. Nunca um sem o outro.

Endpoints (já existem no backend)
Todos autenticados via header Authorization: Bearer ${token}. Base URL = DEFAULT_URL exportado de @/services.

GET    /availability/exceptions          → IExceptionResponse[]
POST   /availability/exceptions          body: IExceptionPayload → IExceptionResponse
DELETE /availability/exceptions/{id}     → 204 No Content

Integração transparente com o calendário público
Não se preocupe com isso. Quando o cliente público (rota /providers/:publicToken/clients/:id/request) busca slots de um dia bloqueado, o endpoint getSignUpSlots já retorna lista vazia automaticamente — você só precisa garantir que o provider consiga criar/listar/remover exceções no painel dele.
